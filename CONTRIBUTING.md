# Contributing

Thank you for your interest in contributing! We welcome contributions from the community. Before you get started, please read this guide carefully.

## Contributor License Agreement

**All contributions require acceptance of our [Contributor License Agreement (CLA)](CLA.md).**

This project is licensed under the AGPL-3.0-or-later. To ensure we can enforce this license and protect the project, we require that all contributors assign copyright of their contributions to the project maintainer, Avery Wagar. This is a standard practice used by the FSF, Apache Foundation, and many other open source organizations.

### How to Sign

Before you open your first pull request you must sign the CLA. This only needs to be done once.

If you are contributing on behalf of your employer or another entity, please ensure you have authorization to sign on their behalf.

## Getting Started

1. Fork the repository
2. Create a feature branch from `main` (`git checkout -b feature/my-feature`)
3. Make your changes
4. Ensure your code passes all existing tests
5. Commit your changes with clear, descriptive commit messages
6. Push to your fork and open a pull request

## Pull Request Guidelines

- Keep PRs focused â€” one feature or fix per PR
- Include a clear description of what your change does and why
- Reference any related issues (e.g., `Closes #42`)
- Make sure CI passes before requesting review
- Be responsive to feedback during code review

## Commit Messages

Write clear, concise commit messages. Use the imperative mood in the subject line:

```
Add support for MIDI input
Fix buffer overflow in signal chain
Update dependencies for PedalKernel 0.3
```

## Reporting Issues

- Check existing issues before opening a new one
- Use the issue templates if available
- Include steps to reproduce, expected behavior, and actual behavior
- Include your environment details (OS, version, relevant hardware)

## Code of Conduct

Be respectful and constructive. We're all here to build something great.

## Questions?

Open a discussion or issue if you're unsure about anything. We'd rather help you contribute than have you give up.
